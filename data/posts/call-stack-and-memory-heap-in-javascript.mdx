## # `Call Stack`과 `Memory Heap`의 역할

[# 목차](#-목차)

- `Call Stack`의 역할

  1. `Primitive(원시)`타입의 데이터를 저장
  2. `Execution Context(실행 컨텍스트)`
  3. `Scope Chain(스코프 체인)`

- `Memory Heap`의 역할

  1. `Reference(참조)`타입의 데이터를 저장

> hoisting, this, function..등의 관리나 실행을 담당하는 `Execution Context(실행 컨텍스트)`나 이를 위해 `{}`간의 유효범위를 위해 상하관계를 정의하는 `Scope Chain(스코프 체인)`의 역할은 다루지 않으며 해당 글에서의 `Call Stack`은 `Primitive(원시)`타입을 저장하는 저장소의 역할을 의미합니다.

## # `Primitive(원시)`타입의 종류와 저장 방식

[# 목차](#-목차)

1.  `Primitive(원시)`타입의 종류 [(JavaScript data Types, MDN Web Docs)](https://developer.mozilla.org/ko/docs/Web/JavaScript/Data_structures)

    - string
    - number
    - bigint
    - boolean
    - null
    - undefined
    - Symbol

2.  `Primitive(원시)`타입의 특징 [(Immutable, MDN Web Docs)](https://developer.mozilla.org/ko/docs/Glossary/Immutable)

    - `Immutable Value(변경 불가능한 값)`

자바스크립트에서는 `Call Stack`과 `Memory Heap`이라는 두 곳의 메모리 공간이 존재합니다.
`Primitive(원시)`타입은 `Call Stack` 한 곳에만 저장되며 경우에 따른 저장 원칙은 다음과 같습니다.

- a라는 변수에 "apple"을 할당하는 경우


3.  변수 a는 식별자로서 해당 주소(A1DC0)를 가리키게 됩니다.

> 즉, 이미 저장된 값("apple")을 변경할 수 없기 때문에 `Call Stack`에 새로운 주소(B2DC5)를 할당하고 값("banana")을 저장하는 것입니다. 그리고 이를 불변성이 유지되었다고 합니다.

## # `Reference(참조)`타입의 종류와 저장 방식

[# 목차](#-목차)

1.  `Reference(참조)`타입의 종류 [(JavaScript data Types, MDN Web Docs)](https://developer.mozilla.org/ko/docs/Web/JavaScript/Data_structures)

    - array
    - object
    - function

2.  `Reference(참조)`타입의 특징 [(Mutable, MDN Web Docs)](https://developer.mozilla.org/ko/docs/Glossary/Mutable)

    - `Mutable Value(변경 가능한 값)`

`Reference(참조)`타입은 `Call Stack`과 `Memory Heap` 두 곳에 모두 저장됩니다.  
`Reference(참조)`타입의 경우, 식별자는 `Call Stack`을 가리키지만 실제 값은 `Memory Heap`에 저장되며 경우에 따른 저장 원칙은 다음과 같습니다.

- alphabet이라는 변수에 "a", "b" 요소를 가진 array타입의 변수 선언


## # 마무리

`Call Stack`의 주소는 값의 비교나 불변성 유지를 판단하는 기준이 됩니다.

- 값을 비교할 때 식별자가 가리키는 주소가 동일하다면 같은 값이고 동일하지 않다면 같은 값이 아니다.
- 값을 변경할 때 식별자가 가리키는 주소가 변경 전의 주소와 다르다면 불변성이 유지되었음을 의미하고, 반면 실제 값은 변경되었지만 주소가 변경 전의 주소와 같다면 불변성이 유지되지 않았음을 의미한다.

`Memory Heap`은 `Primitive(원시)`타입보다 크기가 큰 `Reference(참조)`타입의 값을 저장하기 위한 저장소입니다.
