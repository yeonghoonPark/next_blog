## # Map 객체

[# 목차](#-목차)

`Map` 객체는 클래스 형태로 ECMAScript2015(ES6)에서 도입된 표준 내장 객체<sup>(Standard Built-in Object)</sup>입니다.
키(Key)와 값(value)을 쌍으로 저장하는 자료구조로, 키는 어떠한 데이터 타입도 사용할 수 있으며, 각 키는 유일해야 합니다.
브라우저와 Node.js를 포함하여 자바스크립트가 동작하는 대부분의 환경에서 사용할 수 있으며, 삽입된 순서대로 요소를 순회할 수 있습니다.
`Map`은 *키와 값의 쌍을 효율적으로 관리하고 저장*하는 데 최적화된 컬렉션 자료구조입니다.

---

## # Map 특징

[# 목차](#-목차)

`Map`은 *키와 값을 쌍으로 관리하며 키의 유일성을 보장*하는 자료구조로 다음과 같은 특징이 있습니다.

- _키(Key)로 모든 데이터 타입 허용_  
  ◦ &nbsp; `Map`의 키는 문자열, 숫자, 객체, 함수 등 어떤 타입도 가능하며, 키 자체의 타입과 값이 보존됩니다.

- _키(Key)-값(Value) 쌍 저장 구조_  
  ◦ &nbsp; 값(Value)은 키와 연관되어 저장되며, 프로토타입 메서드를 통해 키를 기준으로 값에 접근할 수 있습니다.

- _삽입 순서 유지_  
  ◦ &nbsp; 요소는 삽입된 순서가 유지되며, 삽입 순서대로 순회됩니다.

- _키(Key)의 중복 불허_  
  ◦ &nbsp; 키(Key)가 중복되면 후에 삽입한 값이 기존값에 덮어씌워집니다.

- _프로토타입 메서드 제공_  
  ◦ &nbsp; `get`, `set`, `has`, `delete`, `clear`, `size` 등 다양한 메서드와 속성을 제공합니다.

- _이터러블(Iterable) 자료구조_  
   ◦ &nbsp; `for...of`, `forEach` 등으로 키-값 쌍을 순회할 수 있습니다.

## # 기본 문법

[# 목차](#-목차)

### 🐣 생성

`Map`은 클래스로, `new` 키워드를 통해 인스턴스를 생성할 수 있습니다.
인자를 전달하지 않을 때는 빈 `Map` 객체가 생성되며,
인자를 전달할 때는 순회할 수 있는<sup>(Iterable)</sup> 키-값 쌍의 객체를 전달해야 합니다.

```javascript
// 빈 Map 생성 (요소 없음)
const map1 = new Map(); // Map(0) {}

// 초기값 전달 생성
const map2 = new Map([
  ["key1", "value1"],
  ["key2", "value2"],
]); // Map(2) {'key1' => 'value1', 'key2' => 'value2'}
```

### ➕ 요소 추가 및 변경

요소를 추가하거나 변경할 때는 `set` 메서드를 사용합니다.
`set(key, value)` 형태로 키와 값을 추가하거나, 이미 키가 존재하는 경우에는 값을 변경합니다.
또한 `set` 메서드는 자신(`Map` 객체)을 반환하므로, 연쇄 호출(Chaining)할 수 있습니다.

```javascript
const map = new Map(); // Map(0) {}

// 키 "a"의 값에 1을 추가
map.set("a", 1); // Map(1) {'a' => 1}

// 키 "a"의 값을 1에서 2로 변경
map.set("a", 2); // Map(1) {'a' => 2}

// 연쇄 호출
map.set("b", 4).set("c", 6); // Map(3) {'a' => 2, 'b' => 4, 'c' => 6}
```

### ➖ 요소 삭제

요소를 삭제할 때는 `delete` 메서드를 사용합니다.
`delete(key)`의 형태로, 인자로 삭제할 키를 전달하여 키-값 쌍을 삭제합니다.
삭제 성공 시에는 `true`, 없으면 `false`를 반환합니다.

```javascript
const map = new Map([
  ["x", 10],
  ["y", 20],
]); // Map(2) {'x' => 10, 'y' => 20}

map.delete("x"); // true
map.delete("z"); // false
```

### 🔍 요소 조회

요소를 조회할 때는 `get`, `has` 메서드를 사용합니다.
`get(key)` 메서드는 키에 대응하는 값을 조회하며, 키가 없으면 `undefined`를 반환합니다.
`has(key)` 메서드는 키가 존재하면 `true`, 아니면 `false`를 반환합니다.

```javascript
const map = new Map([
  ["x", 10],
  ["y", 20],
]); // Map(2) {'x' => 10, 'y' => 20}
map.get("x"); // 10
map.get("z"); // undefined

map.has("x"); // true
map.has("z"); // false
```

### 🔄 순회

`Map` 객체는 순회할 수 있는 자료구조로,
삽입된 순서대로 `for...of` 또는 `forEach` 메서드를 통해 순회할 수 있습니다.
`index` 프로퍼티는 존재하지 않지만, 요소의 삽입 순서가 보장되며,
각 반복에서 `[key, value]`를 순회합니다.

```javascript
const map = new Map([
  ["x", 10],
  ["y", 20],
]); // Map(2) {'x' => 10, 'y' => 20}

// 1. for...of
for (const [key, value] of map) {
  console.log(key, value);
}

// 2. forEach (value, key, map)
map.forEach((value, key) => {
  console.log(key, value);
});
```

### 🥾 크기 확인

저장된 키-값 쌍의 개수는 인스턴스의 `size` 프로퍼티를 통해 확인합니다.

```javascript
const map = new Map([
  ["x", 10],
  ["y", 20],
]); // Map(2) {'x' => 10, 'y' => 20}

map.size; // 2
```

## # Map과 Object의 차이

[# 목차](#-목차)

`Map`과 `Object`는 *키와 값을 쌍으로 저장하는 자료구조*라는 공통점을 가지고 있지만,
키 타입과 속성 보존, 순서 관리, 크기 확인 등에서 중요한 차이가 있습니다.

따라서 `Map`은 특히 동적으로 키의 타입이 다양하거나,
키 타입을 보존해야 하며, 삽입 순서와 크기를 빠르게 관리해야 하는 상황에 적합합니다.
반변, `Object`는 정적인 구조나 _JSON_ 같이 단순하고 명확한 데이터 표현에 효율적이며,
`Prototype` 상속 기능이 필요한 객체 모델을 구현할 때 유용합니다.

## # 편의성

[# 목차](#-목차)

`Map` 객체는 `Object`에 비해 여러 편의성을 제공합니다.
특히 키 타입 보존, 크기 확인, 다양한 메서드 지원 등에서 효율적이고 코드 가독성 및 유지보수성 측면에서 장점을 가집니다.

- _명확한 크기 관리_  
  `Object`는 길이를 확인하기 위해 `Object.keys(obj).length` 같은 수동적 변환이 필요하지만,
  `Map`은 `size` 프로퍼티를 통해 즉시 크기를 확인할 수 있습니다.

- _순수한 키 관리_  
  `Object`는 기본적으로 `prototype` 체인을 따라 다양한 상속이 포함되기 때문에,
  실제 데이터만을 관리하려면 별도의 보정이 필요하지만, `Map`은 순수하게 지정한 키만을 관리합니다.

- _가독성 및 체이닝_  
  `set`, `get`, `has`, `delete` 등의 직관적인 메서드를 지원하며,
  `set` 메서드는 `Map` 자체를 반환하기 때문에 체이닝이 가능해 여러 요소를 직관적으로 추가하여 관리할 수 있습니다.

- _순회 방식의 다양성_  
  `for...of`, `forEach` 외에도
  `keys`, `values`, `entries` 메서드를 통해 원하는 순회 방식을 상황에 따라 선택하여 활용할 수 있습니다.

## # 함정

[# 목차](#-목차)

`Map`은 분명한 장점이 많지만, 모든 상황에서 최적의 선택은 아닙니다.
특히 주의할 점들이 있어 상황에 따라 적절함을 판단하여 사용해야 합니다.

### 🤔 불필요한 사용

- 데이터 키가 문자열(String) 또는 심볼(Symbol)로 제한되고,
  데이터 양이 많지 않으며, JSON 변환이 필요한 단순한 구조라면 `Object`를 사용하는 것이 더 간편하고 직관적입니다.

- `Map`은 삽입 순서가 보장되고 다양한 편의성이 높은 메서드를 제공하는 강점이 있지만,
  단순 키-값 맵핑 정도라면 오히려 불필요한 오버헤드가 될 수 있습니다.
  간단한 데이터 구조라면 `Object` 생성 비용이 상대적으로 저렴하고 메모리 사용량도 적습니다.

### 🧨 직렬화 및 변환의 불편함

- `Map`은 기본적으로 JSON 직렬화(`JSON.stringify`)가 불가능합니다.

- 직렬화가 필요한 경우, 별도의 변환 과정을 거쳐야 합니다.
  `Array.from` 혹은 반복문을 통해 `[key, value]` 형식의 배열로 변환 뒤에 JSON 처리가 가능합니다.

```javascript
const map = new Map([
  ["a", 1],
  ["b", 2],
]);

// 직렬화 전 변환 필요
const serializedMap = JSON.stringify(Array.from(map)); // [["a",1],["b",2]]
```

### 📌 성능 고려

- `Map`은 키 기반 탐색에서 평균적으로 _O(1)_ 의 성능 이점을 제공하지만, 모든 작업에 항상 빠른 것은 아닙니다.
  단순 순회를 목적으로 한다면 `Array`와 동일한 _O(n)_ 시간 복잡도가 필요합니다.
  따라서 *빠른 키 기반 검색*이 필요한 경우에 가장 이점을 가집니다.

## # 마무리

[# 목차](#-목차)

_# references_

자바스크립트 `Map` 객체는 다양한 타입의 키를 보존하면서 키-값 쌍을 효율적으로 관리할 수 있는 자료구조입니다.
특히 `Object`에 비해 _삽입 순서 유지_, _크기 확인_, _프로토타입 속성과 무관_ 등 직관적이고 투명한 데이터 관리 방식을 제공합니다.
그러나 단순 데이터 모델이나 JSON 직렬화가 필요한 상황에서는 `Object`가 여전히 더 적합할 수 있습니다.
즉, 상황에 따라 `Map`과 `Object`를 올바르게 선택하는 것이 중요합니다.
`Map`은 키 관리가 유연하고,
효율적인 키 탐색을 지원하므로 동적인 데이터 관리와 다양한 키 타입을 활용하여 빠른 조회 성능이 요구되는 상황에서 빛을 내는 객체입니다.
