## 자바스크립트의 데이터타입과 그에 따른 저장 방식

> `Primitive` & `Reference`

자바스크립트에서 데이터는 크게 `Primitive`와 `Reference` 타입으로 나누어집니다.  
이 두 종류 타입의 데이터는 변수에 할당할 수 있으며 변수에 할당된 데이터는 메모리에 저장됩니다, 하지만 메모리에 저장되는 방식에 있어 두 종류 타입의 데이터는 서로 차이점이 존재합니다.  
이 차이점을 이해한다면 보다 효율적인 코드를 작성하는 데 도움이 되리라 생각합니다.

## # `Primitive`타입의 종류와 저장 방식

-   `Primitive`타입의 종류 : _string_, _number_, _boolean_, _null_, _undefined_, _Symbol_

자바스크립트에서는 `Call Stack`과 `Memory Heap`이라는 두 곳의 메모리 공간이 존재합니다.
`Primitive`타입으로 변수를 선언하고 값을 할당할 경우, 오직 `Call Stack`에서만 값이 저장되며 `Primitive`타입을 변수로 선언하여 값을 할당, 호출, 재할당시에 일어나는 일에 대해 알아보겠습니다.

<br />

1.  a라는 변수에 "apple"을 할당하는 경우

```typescript
let a = "apple";
```

-   `Call Stack`의 메모리 공간을 확보합니다, 이 확보된 메모리 공간은 주소를 가지게 되며 이곳에 "apple"이라는 값을 저장하고 변수 a는 식별자로서 해당주소를 가리키고 있습니다.

---

2.  a를 호출하는 경우

```typescript
a;
```

-   a가 `Call Stack`의 메모리 공간에서 가리키고 있는 주소를 찾아 저장되어 있는 값 "apple"을 꺼내옵니다.

---

3. a의 값을 "banana"로 재할당하는 경우

```typescript
a = "banana";
```

-   `Call Stack`에서 처음 "apple"을 할당받을 때의 주소가 아닌 새로운 주소를 가지게 되며 그곳에 "banana"를 저장하고 식별자 b는 해당 주소를 가리키게 됩니다.

4. b라는 변수에 "banana"를 할당하는 경우

```typescript
let b = "banana";
```

-   `Call Stack`의 메모리에 이미 "banana"라는 값이 존재하므로 새로운 주소에 저장하는 것이 아닌 기존의 "banana"가 저장된 주소를 재사용하며 b는 식별자로서 해당 주소를 가리키게 됩니다.

<br />

> a와 b라는 변수가 최종적으로 의미하는 것은 "apple"이나 "banana"라는 값이라기보다 식별자로서 `Call Stack`에서 가리키고 있는 주소를 의미하며 호출될 경우, 단지 그 시점의 해당주소에 저장되어 있는 값을 꺼내올 뿐입니다.
