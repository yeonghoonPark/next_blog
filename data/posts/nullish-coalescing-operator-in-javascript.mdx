## # null 병합 연산자의 구조

[# 목차](#-목차)

1. 구조

```typescript
leftExpression ?? rightExpression;
```

- 2개의 물음표(??)와 2개의 피연산자<sup>(expression)</sup>으로 구성되어 있습니다.
- 왼쪽은 왼쪽 피연산자, 오른쪽은 오른쪽 피연산자 칭합니다.

---

## # null 병합 연산자의 원리

[# 목차](#-목차)

1. 원리

```typescript
(null || undefined) ?? rightExpression;
```

- 왼쪽 피연산자의 값이 null 또는 undefined인 경우 오른쪽이 실행되고 그렇지 않다면 왼쪽이 실행됩니다.

---

## # null 병합 연산자의 예시

[# 목차](#-목차)

1.  예시

```typescript
let something;

const printEmpty = () => {
  console.log("empty");
};

something ?? printEmpty();
```

- "something"은 선언은 되었지만 값이 할당되지 않았기 때문에 undefined입니다.
- 화살표 함수 "printEmpty"는 console에 "empty"를 출력하는 함수입니다.
- 왼쪽 피연산자가 undefined이기 때문에 오른쪽 피연산자인 함수 "printEmpty"를 실행합니다.

<br />

2.  if문 변경 예시

```typescript
// if...else
const printMessage = (text) => {
  let message;
  if (text === null || text === undefined) {
    message = "empty";
  } else {
    message = text;
  }
  console.log(message);
};

// nullish coalescing operator
const printMessage = (text) => {
  let message = text ?? "empty";
  console.log(message);
};
```

- if문에 비해 코드를 간결하게 작성할 수 있습니다.

## # 논리적 OR 연산자와의 차이점

[# 목차](#-목차)

1. 차이점

```typescript
falsy || rightExpression;
```

- 왼쪽 피연산자가 falsy인 경우 오른쪽이 실행되고 그렇지 않다면 왼쪽이 실행됩니다.

2.  falsy가 되는 경우

    - null
    - undefined
    - false
    - 0
    - -0
    - NaN
    - "" ('', ``)
