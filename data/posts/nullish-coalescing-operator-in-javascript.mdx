## # `nullish coalescing operator(null 병합 연산자)`의 구조

[# 목차](#-목차)

1. 구조

```typescript
leftExpression ?? rightExpression;
```

- 2개의 물음표(??)와 2개의 expression(피연산자)으로 구성되어 있습니다.
- 왼쪽은 left expression, 오른쪽은 right expression이라고 칭합니다.

---

## # `nullish coalescing operator(null 병합 연산자)`의 원리

[# 목차](#-목차)

1. 원리

```typescript
(null || undefined) ?? rightExpression;
```

- left expression의 값이 `null` 또는 `undefined`인 경우 오른쪽이 실행되고 그렇지 않다면 왼쪽이 실행됩니다.

---

## # `nullish coalescing operator(null 병합 연산자)`의 예시

[# 목차](#-목차)

1.  예시

```typescript
let something;

const printEmpty = () => {
  console.log("empty");
};

something ?? printEmpty();
```

- something은 선언은 되었지만 값이 할당되지 않았기 때문에 `undefined`입니다.
- printEmpty()는 console에 "empty"를 출력하는 함수입니다.
- left expression이 `undefined`이기 때문에 right expression인 printEmpty()를 실행합니다.

<br />

2.  if문 변경 예시

```typescript
// if...else
const printMessage = (text) => {
  let message;
  if (text === null || text === undefined) {
    message = "empty";
  } else {
    message = text;
  }
  console.log(message);
};

// nullish coalescing operator
const printMessage = (text) => {
  let message = text ?? "empty";
  console.log(message);
};
```

- if문에 비해 코드를 간결하게 작성할 수 있습니다.

## # `logical OR operator(논리적 OR 연산자)`와의 차이점

[# 목차](#-목차)

1. 차이점

```typescript
falsy || rightExpression;
```

- left expression이 `falsy`인 경우 오른쪽이 실행되고 그렇지 않다면 왼쪽이 실행됩니다.

2.  `falsy`가 되는 경우

    - null
    - undefined
    - false
    - 0
    - -0
    - NaN
    - "" ('', ``)

## # 마무리

    1.  `nullish coalescing operator(null 병합 연산자)`

        -   null
        -   undefined

    2.  `logical OR operator(논리적 OR 연산자)`

        -   null
        -   undefined
        -   false
        -   0
        -   -0
        -   NaN
        -   "" ('', ``)

    3.  `default parameter`

        -   undefined

각 연산자의 정확한 차이점을 인지하지 못해 예상치 못한 결과를 마주하고 머리를 긁적인 적이 있습니다. 해당 글을 통해 각 연산자의 구조와 원리를 이해하는 데 조금이나마 도움이 됐으면 좋겠습니다.
